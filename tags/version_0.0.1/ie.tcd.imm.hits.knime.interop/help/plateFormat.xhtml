Plate Format

Overview

This node converts between different plate formats. It support 96, 384 and 1536 well count plates.

Sample workflow can be found here.

The node description shows the overview of the functionality, while this page gives more detailed examples.

Input data

It expects exactly one input table.

First data inport

The input table should contain the following columns (mind the case of letters):

  • Plate, or plate
    • Integral type
    • starting from 1,
  • col
    • Integral type
    • starting from 1,
  • row
    • Integral type
    • starting from 1.
All other columns remain the same, the content of these columns will change according to the settings.

Hint: You might want to try the BioConverter node to create this kind of table.

Output data

It has only one output table.

First data outport

Same format as the data.

Hint: You might want to try the BioConverter node to modify the table to your needs.

Options

Input well count

The number of wells on input plates.

Output well count

The number of wells on plates to generate.

Layout
