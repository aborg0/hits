Use cases

Initial steps

You need the following nodes:

  1. Importer (or equivalent)
  2. Simple Configurator for cellHTS2 (or two equivalent)
  3. File Reader to read a file with proper header
  4. cellHTS2

To reorder the columns use Column Resorter node.

Connect the nodes like the following picture shows (connect Importer to first input port of CellHTS2, Simple Configurator for CellHTS2 first and second to CellHTS2's second and third input ports, and the File Reader to the fourth input port of CellHTS2):

Data visualisation

You may visualise more than one experiment or normalisation type if you append one to an another. Use Concatenate node to get this behaviour. (If you want to get the values comparable in normal nodes the Joiner node is a good choice.)

Check the data visually more attracting. The image below shows a possible layout:

You may add a RowID in front of Scatter Plot and Paralell Coordinates to add more meaningful tooltips to the points.

You may add a Color Manager in front of the visual nodes to add information (like well content type (sample, positive control, ...)) about the points.

The other visualisation nodes, like Histogram (interactive) are also benefit from the extra colour information, and also give meaningful data.

The order of the parameters follows the add order (add always appends to the end).

You can select/filter out the HiLited rows using the HiLite Filter if it is a sibling node.

You may also examine you data in different normalisation parameters/experiments using the Heatmap node, if you generate the different normalised data for all experiments and Concatenate the result.

Hits by rank

Use the Rank node followed by a Row Filter node. (You may use another Row Filter to filter out the non-sample wells.)

Hits by cut-off points

Use the Row Filter to select the proper rows. (Ex.: select the rows if the score of a given parameter is not in the [-2,2] interval.)

You may select hits satisfying multiple parameter criteria by appending filters one after another, or select the hits for some parameters and than Concatenate them.

Hits by interquartile ranges

The RowID node is optional, it is there only to get better tooltips.

You can select only the really outliers (outside of the whiskers), but in the output you can some further statistics.

Clustering

Shows a simple clustering setup.

For some of the clustering algorithms you have to filter out the non-data columns, and add a predictor to assign the rows to the proper cluster.

Export data/hits

You can export your data to either xls Excel format or either comma separated values.

Compare with other experiments

If you append the columns to the original experiments you can view the results in one row, plot against the two experiments. (You should try to append o suffix for column names. Some will sound as an Italian word. ;))

When you want separate rows for the experiments just Concatenate them.
