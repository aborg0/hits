Importer node overview

With this node you can read IN Cell Analyzer 1000 data in xls format. The node description contains the basics, this page shows the details.

Configuration

In The folder of xls files: group select the folder of xls files. Using the [Browse] button you may select an individual xls file also. Selection of an xls file will actually select the folder of it.

You can rearrange the files in the xls list using the [^] and [v] buttons.

It is important to select the xls files you want to analyse. By default none of them is selected.

The files are grouped by the Replicate count to create the replicates. So if you have 3 replicates, then the forth xls file will belong to the second plate group and the first replicate group.

You have to select the proper Well count and also the Plate count.

Optionally you can select an annotation file to have the data and your annotation together.

Expected xls file format

There must be a (the second) sheet with name: Summary by wells.

The first column should contain the well identifiers in format: A - 1 from the 4th row.

The header of the data is obtained from the second row, from the 3rd column.

The data starts from 4th row in the 3rd column.

Expected annotation file format

This is a tab separated file with Plate, Well, GeneID, GeneSymbol header.

The plate numbering should start from 1.

The format of values in the Well column is like: A01.

For more information consult the cellHTS2 complete manual, Probe annotation section.

If there are more annotation than input data, then you will get an error.

Output

Structure

  • Plate
  • Replicate
  • Well
  • Parameters - floating point number (Double) type, every parameter column in the xls file will be present here in the same order.
  • GeneID - optional, only if annotation file is selected
  • GeneSymbol - optional, only if annotation file is selected
