Simple Configurator for cellHTS2 node overview

With this node you can read the plate configuration file of cellHTS2, and also the description file. The node description shows the overview of the functionality, while this page gives a more detailed explanation.

Configuration

Select the plate configuration file in the first area, and the description file in the second.

The default filter will show only the Plateconfig.txt/Description.txt files at first choice, but you can easily select the .txt filter.

Expected plate configuration file format

This is a tab separated file. The first row should contain the Wells: 96 text, while the second: Plates: 1 with your plate number. For further details see the cellHTS2 complete manual, 4.1 (Screen configuration: annotating the plate results/Format of the plate configuration file)

If the plates are not in synchron with the data you will get error in the cellHTS2 node.

Expected annotation file format

This is a normal text file. The content will be loaded in a structured manner.

For more information consult the cellHTS2 complete manual, 4 Screen configuration: annotating the plate results section.

Output

First output port

This is the plate configuration content. Compatible with cellHTS2 node second input port.

Structure

  • Plate - text (String) type; it differs from the normal plate columns, because this may contain patterns.
  • Well - text (String) type; it differs from the normal well columns, because this may contain patterns.
  • Content - text (String) type; Usually sample, pos, neg, other, empty content is here. This is case insensitive.

Second output port

This is the description content. Compatible with cellHTS2 node third input port.

Structure

  • Group - text (String) type; the group of the key.
  • Key - text (String) type; a key for the value.
  • Value - text (String) type; the value for the group-key pair.
