Columns

  • Experiment - text (String) type, contains the experiment name
  • Normalisation method - text (String) type, contains the normalisation method, possible terms:
    • Bscore
    • POC
    • negatives
    • NPI
    • mean
    • shorth
    • locfit
    • loess
  • log transform - text (String) type, terms: the log text, or empty
  • Normalisation kind - text (String) type, the normalisation kind (multiplicative or additive) are the possible values
  • Variance adjustment - text (String) type, variance adjustment, possible terms: no adjustment, by plate and by experiment
  • Scoring method - text (String) type, scoring method, possible terms:
    • none
    • zscore
    • NPI
    • zscoreByPlate
    • zscoreNonRobust
    • zscoreNonRobustByPlate
  • Summarise method - text (String) type, the summarisation method, possible terms:
    • mean
    • median
    • min
    • max
    • closestToZero
    • furthestToZero
    • rms
  • Plate - integral (Integer) type, starting from 1, the plate values
  • Position - integral (Integer) type, starting from 1, the position on the plate, A1 is 1, A12 is 12
  • Well - text (String) type, the well identifier, format like: A01
  • Replicate - integral (Integer) type, starting from 1 the replicate values
  • final well annotation - text (String) type, this is the well annotation after applying the screen log file; some wells may be flagged; it may be multiple, depending on the parameters, but it can be on its own (in that case it gets a _null suffix)
  • well annotation - text (String) type, this is the original well annotation like sample, pos, neg, other
  • Gene id - text (String) type, an identifier of the gene
  • Gene symbol - text (String) type, the gene symbol
  • Gene annotation - text (String) type, additional gene annotations (any other textual column from the input)
  • empty - text (String) type, empty column
  • score - floating point number (Double) type, does not depend on replicates, but depends on parameters these are the scored values
  • mean or diff - floating point number (Double) type, does not depend on replicates, but depends on parameters; when there are two replicates the result is the raw value from the second replicate minus raw value of the first replicate, otherwise the mean of the replicate values
  • median - floating point number (Double) type, does not depend on replicates, but depends on parameters, the median of the raw replicate values
  • raw - floating point number (Double) type, depends both on replicates and parameters. These are the original values
  • normalised - floating point number (Double) type, depends both on replicates and parameters. These are the normalised values
  • raw/(plate, replicate mean) - floating point number (Double) type, depends both on replicates and parameters. These are the raw value divided by the physical plate mean

Column order

In this panel you can set the column order and select which one should be present in the output.

There are four special columns, the

  • group by channels start
  • group by channels end
  • group by replicates start
  • group by replicates end

These entries allows you to group the results to your need, but you must have both if the specified columns depend on both channels and replicates too, and only the channels if it depends on the channels. You may have more then one channel in the start-end area, but the start-end intervals may not overlap.

Colours

You can set which data should have which colour. Currently you have only one choice for each data option.

Connection

The proxy settings are important if you are behind a firewall and want to use nodes accessing services outside of your network.
